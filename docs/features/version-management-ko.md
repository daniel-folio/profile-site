# 🎨 디자인 버전 관리 시스템 (Design Version Management)

이 포트폴리오 웹사이트는 **다양한 테마와 UI 디자인 버전을 실시간으로 안전하게 전환**할 수 있는 강력한 '버전 스위칭 (Version Switching)' 시스템을 갖추고 있습니다.

본 문서는 디자인의 격리된 구조와 Strapi를 이용한 실시간 전환 방법, 그리고 미래의 확장이 얼마나 쉬운지 상세히 설명합니다.

---

## 📂 1. 프론트엔드 폴더 구조 (Folder Structure)

**"디자인 컴포넌트는 오직 자신이 해당하는 버전 폴더에만 격리되어 존재합니다."** 

공통으로 쓰이는 컴포넌트(로그인, 공통 버튼 등)만 `common`에 두고, 나머지 디자인 요소는 철저히 버전별로 분리하여 관리합니다. 이를 통해 v1과 v2의 CSS나 코드가 서로 섞여 망가지는 현상을 완벽하게 방지합니다.

```plaintext
frontend/src/
├── app/                  # 화면 분기점 (페이지 컨트롤러)
│   ├── page.tsx          # 메인 화면 (V1/V2 중 무엇을 보여줄지 결정)
│   ├── resume/page.tsx   # 이력서 화면
│   └── career-detail/page.tsx # 경력 상세 화면
│
└── components/           # 디자인 요소들
    ├── common/           # 공용 (모든 버전에 공통으로 쓰이는 버튼, 인풋 등)
    ├── v1/               # 버전 1 (클래식 / 라이트 모드 디자인)
    │   ├── layout/       # v1 전용 헤더 & 푸터
    │   ├── pages/        # v1 전용 화면들 (HomePageClientV1 등)
    │   └── sections/     # v1 전용 작은 블록들 (Hero, Projects 등)
    │
    └── v2/               # 버전 2 (모던 / 다크 / 파티클 배경 디자인)
        ├── layout/       
        ├── pages/        
        ├── sections/     
        └── styles/       # v2 전용 스타일 (globals-v2.css)
```

## ⚙️ 2. 버전 전환 방식 (How to Switch)

라이브 서버를 중단하거나 코드를 다시 배포할 필요 없이, Strapi 어드민에서 텍스트 입력만으로 환경을 즉각 변경할 수 있습니다.

1. **Strapi 관리자 패널**(`http://localhost:1337/admin`)에 로그인합니다.
2. 좌측 메뉴에서 **Content Manager** -> **Site Settings** (단일 타입)에 들어갑니다.
3. **`Portfolio Version`** 필드를 찾아 원하는 버전 키를 입력합니다.
   - `v1` 입력 시: **클래식 UI**가 송출됩니다.
   - `v2` 입력 시: **모던 파티클 UI**가 송출됩니다.
4. **Save(저장)** 클릭 후, 반드시 **Publish(게시)** 버튼을 눌러야 데이터가 실제 사이트에 적용됩니다.
5. 사이트를 새로고침하면 디자인이 즉각 변경된 것을 확인할 수 있습니다.

## 🏗️ 3. 작동 원리 (쉽게 읽기)

기존에는 "V2면 V2를 보여줘"라는 `if/else` 분기문을 사용했지만, 이제는 **"버전 선택기(Map)"**라는 더 정석적인 방식을 사용합니다.

* **맵(Map) 구조**: `{ v1: 화면1, v2: 화면2 }`와 같이 버전 이름과 실제 화면을 미리 짝지어둔 '메뉴판'을 사용합니다.
* **실시간 반영**: `force-dynamic` 설정을 통해 서버가 매번 백엔드의 최신 설정값을 확인합니다. 덕분에 서버를 껐다 켤 필요가 없습니다.
* **똑똑한 주소 찾기**: 개발용(Local)과 실제 서버(Test/Production)의 주소를 스스로 판단하여 접속하므로, 어디서든 끊김 없이 동작합니다.

---

## 🛠️ 4. 미래 확장성 (V3 추가하기)

미래에 새로운 **`v3`** 디자인이 추가되더라도, **기존의 v1, v2 코드는 단 한 줄도 수정할 필요가 없습니다.**

1. `src/components/v3/` 폴더 트리를 만들고 새 디자인을 개발합니다.
2. 각 페이지의 "버전 선택기" 맵에 `v3: PageV3` **딱 한 줄만 추가**하면 모든 연동이 끝납니다.
3. 이 방식은 기존 코드를 건드리지 않으므로 예기치 않은 버그가 발생하지 않으며, 언제든 1초 만에 이전 버전으로 되돌릴 수 있는 안전한 구조입니다.

> [!IMPORTANT]
> **배포 시 주의사항**  
> 새로운 서버에 배포한 후 디자인이 바뀌지 않는다면, Strapi의 **Public 권한**에서 `site-setting`의 `find` 권한이 체크되어 있는지 꼭 확인하세요! 권한이 없으면 프론트엔드에서 데이터를 읽어오지 못해 기본값(`v1`)으로만 나오게 됩니다.
